Based on the video, here is a breakdown of what happens when switching between the service type filters:

1.  **How the category chips change:** When the user switches from the 'In Store' filter to the 'Comes to You' filter, the horizontal list of category chips resets its scroll position back to the beginning (starting with 'All', 'Hair', 'Nails', etc.). However, the actual categories available in the list do not appear to change or update.
2.  **What is shown vs. what should be shown:** While the 'In Store' filter is active, the user scrolls to reveal categories like 'Tattoo & Piercing', 'Teeth Whitening', and 'Tanning'. When they switch to 'Comes to You', the list resets, but it appears to be the same master list of categories. Ideally, when 'Comes to You' is selected, the category list should dynamically update to only show services that are actually offered as mobile services, hiding categories that are strictly in-store.
3.  **Filtering inconsistencies:** The primary inconsistency is that the category filter list is static and independent of the selected service type. Because the list doesn't update, a user could potentially have 'Comes to You' selected while still being able to filter by an exclusively in-store category like 'Tattoo & Piercing', which would likely lead to zero search results or a confusing user experience.
